# Agent Skills 集合

这是一个包含多种专业技能的仓库，为 AI 助手提供特定领域的知识和工具。每个 skill 都专注于解决特定类型的问题，帮助用户更高效地完成任务。

## 项目结构

```
skills/
├── cpp-coding-standards/      # C++ 编码规范
├── docx/                      # Word 文档处理
├── encoding-converter/        # 文件编码转换
├── find-skills/               # 技能发现与安装
├── function-refactoring/      # 函数重构
├── patent-application-creator/# 专利申请创建
├── patent-architect/          # 中文专利申请表生成
├── patent-claims-analyzer/    # 专利权利要求分析
├── patent-drafting/           # 专利申请撰写
├── patents-search/            # 专利搜索
├── pdf-vision-reader/         # PDF 视觉解析
├── prior-art-search/          # 现有技术检索
├── vercel-react-native-skills/# React Native 最佳实践
└── web-design-guidelines/     # Web 设计规范
```

## Skills 列表

### 1. cpp-coding-standards (C++ 编码规范)

基于 C++ Core Guidelines 的现代 C++ (C++17/20/23) 编码规范。适用于编写、审查或重构 C++ 代码时强制执行现代、安全和惯用的实践。

**核心原则**：
- RAII 资源管理
- 默认不可变性
- 类型安全
- 表达意图清晰
- 最小化复杂度

**适用场景**：
- 编写新的 C++ 代码
- 审查或重构现有 C++ 代码
- 在 C++ 项目中做出架构决策
- 在语言特性之间做出选择

---

### 2. docx (Word 文档处理)

创建、读取、编辑和操作 Word 文档 (.docx 文件) 的技能。

**主要功能**：
- 使用 `docx-js` 创建新文档
- 解压 → 编辑 XML → 重新打包的工作流编辑现有文档
- 支持表格、图片、超链接、脚注、目录等
- 跟踪修订和注释处理
- 文档验证和修复

**适用场景**：
- 生成报告、备忘录、信函、模板
- 提取或重组 .docx 文件内容
- 插入或替换文档中的图片
- 查找和替换操作

---

### 3. encoding-converter (文件编码转换)

在 GBK、UTF-8、UTF-8 BOM 等编码之间转换文件编码。解决项目中的编码问题、乱码字符或编译错误。

**支持编码**：
- UTF-8 (无 BOM) - 推荐用于跨平台项目
- UTF-8 with BOM - Windows 默认
- GBK/GB2312 - 中文 Windows 传统编码
- UTF-16 LE

**适用场景**：
- 解决 Qt/C++ 项目中的混合编码问题
- 为需要 BOM 的编辑器添加 BOM
- 为遗留系统转换 UTF-8 到 GBK

---

### 4. find-skills (技能发现与安装)

帮助用户发现和安装 agent skills。当用户想要扩展 AI 助手的能力时使用此技能。

**主要功能**：
- 使用 `npx skills find` 搜索可用技能
- 使用 `npx skills add` 安装技能
- 检查和更新已安装的技能

**适用场景**：
- 用户问"怎么做某事"
- 用户说"找一个能做某事的技能"
- 用户想扩展 AI 助手的能力

---

### 5. function-refactoring (函数重构)

重构过长函数，拆分为职责单一的小函数。适用于函数超过100行、圈复杂度过高、难以测试和维护的场景。

**重构流程**：
1. 评估函数质量
2. 识别逻辑分段
3. 提取上下文结构体
4. 拆分子函数
5. 消除魔数
6. 重构主函数

**核心原则**：一个函数只做一件事，每个子函数控制在50行以内。

---

### 6. patent-application-creator (专利申请创建)

从发明披露到 USPTO 就绪申请文件的端到端专利申请创建。

**完整工作流程**：
1. 现有技术检索
2. 权利要求撰写
3. 说明书撰写
4. 图表生成
5. 摘要创建
6. 合规检查
7. IDS 准备

**适用场景**：
- 创建完整的专利申请
- 起草临时专利申请
- 准备实用新型专利申请
- 撰写专利权利要求和说明书

---

### 7. patent-architect (中文专利申请表生成)

将技术创意转化为完整的中国专利申请表 (专利申请表)。自动通过 SerpAPI 检索现有技术。

**执行阶段**：
1. 理解发明 - 提取核心技术要素
2. 现有技术检索 - API 搜索和 WebSearch 回退
3. 生成申请表 - 按模板起草完整文档

**输出**：符合中国专利申请格式的完整文档，包括技术领域、技术问题、技术方案、技术效果等。

---

### 8. patent-claims-analyzer (专利权利要求分析)

自动分析专利权利要求的 USPTO 合规性，检查 35 USC 112(b) 要求。

**分析内容**：
- 前提基础检查 - 发现未经引入就使用的术语
- 明确性分析 - 识别主观/不明确术语
- 权利要求结构验证 - 解析独立/从属权利要求
- 问题分类 - 关键/重要/轻微

**输出**：合规评分 (0-100)、问题列表、MPEP 引用、具体修复建议。

---

### 9. patent-drafting (专利申请撰写)

起草专利申请、撰写权利要求、分析现有技术或回应审查意见。

**参考文件**：
- `references/patterns.md` - 创建模式指导
- `references/sharp_edges.md` - 关键失败原因
- `references/validations.md` - 验证规则和约束

**适用场景**：
- 撰写专利申请
- 权利要求策略
- USPTO 实践指导

---

### 10. patents-search (专利搜索)

使用自然语言查询搜索全球专利数据库，由 Valyu 语义搜索 API 提供支持。

**特点**：
- 无需解析 API 参数 - 直接传入自然语言查询
- 语义搜索 - 理解查询含义而非关键词匹配
- 全文访问 - 返回完整专利信息
- 图片链接 - 包含专利图表和附图

**适用场景**：
- 现有技术检索和专利景观分析
- 技术趋势分析
- 竞争对手创新跟踪
- 专利自由实施分析

---

### 11. pdf-vision-reader (PDF 视觉解析)

将 PDF 页面转换为图像，使用视觉分析提取内容包括图表、图形和视觉元素。适用于富含视觉内容的 PDF。

**工作流程**：
1. 使用 `pdf_to_images.py` 将 PDF 转换为图像
2. 逐页读取和分析图像
3. 整合为 Markdown 格式

**提取内容**：
- 文本信息（标题、正文、列表）
- 图表（流程图、组织图等）
- 图形和图表数据
- 表格（转换为 Markdown 表格）
- 布局和结构

---

### 12. prior-art-search (现有技术检索)

使用 BigQuery 和 CPC 分类的系统化 7 步方法论，进行全面的专利现有技术检索和可专利性评估。

**7 步方法论**：
1. 发明定义 - 提取关键特征
2. 关键词策略 - 开发搜索词列表
3. 广泛关键词搜索 - 在 BigQuery 上运行
4. CPC 代码识别 - 找到相关分类代码
5. 深度 CPC 搜索 - 在分类内全面搜索
6. 时间线分析 - 理解技术演进
7. 可专利性报告 - 专业评估和建议

**输出**：包含可专利性评估、前 10 个相关现有技术、权利要求策略建议、IDS 列表的完整报告。

---

### 13. vercel-react-native-skills (React Native 最佳实践)

React Native 和 Expo 应用开发的全面最佳实践指南。

**规则分类（按优先级）**：
1. **列表性能（关键）** - 使用 FlashList、记忆化列表项、稳定回调引用
2. **动画（高）** - 仅动画 transform 和 opacity、使用 useDerivedValue
3. **导航（高）** - 使用原生导航器而非 JS 导航器
4. **UI 模式（高）** - 使用 expo-image、Pressable、原生菜单
5. **状态管理（中）** - 最小化状态订阅、dispatcher 模式
6. **渲染（中）** - 文本包装在 Text 组件中
7. **Monorepo（中）** - 原生依赖放在 app 包中
8. **配置（低）** - 字体配置插件、设计系统导入组织

---

### 14. web-design-guidelines (Web 设计规范)

审查 UI 代码是否符合 Web Interface Guidelines。用于检查可访问性、审计设计、审查 UX。

**工作流程**：
1. 从源 URL 获取最新指南
2. 读取指定文件
3. 根据获取的指南检查所有规则
4. 以简洁的 `file:line` 格式输出发现

**适用场景**：
- 用户要求"审查我的 UI"
- 检查可访问性
- 审计设计
- 检查网站是否符合最佳实践
