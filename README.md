# Agent Skills 集合

这是一个包含多种专业技能的仓库，为 AI 助手提供特定领域的知识和工具。每个 skill 都专注于解决特定类型的问题，帮助用户更高效地完成任务。

## 项目结构

```
skills/
├── cpp-allman-style/          # C++ Allman 风格规范
├── cpp-coding-standards/      # C++ 编码规范
├── dev-plan-manager/          # 开发计划管理器
├── docx/                      # Word 文档处理
├── encoding-converter/        # 文件编码转换
├── find-skills/               # 技能发现与安装
├── function-refactoring/      # 函数重构
├── planning-and-task-breakdown/# 规划和任务分解
├── requirements-analysis/     # 需求分析
└── task-planning/             # 任务规划
```

## Skills 列表

### 1. cpp-allman-style (C++ Allman 风格规范)

C++ 代码风格规范，使用 Allman 风格（花括号独占一行）。当编写或修改 C++ 代码时自动应用此规范。

**核心规则**：
- 左花括号独占一行，与语句对齐
- Lambda 表达式：短则紧凑，长则换行
- 单行语句可省略花括号

**适用场景**：
- 编写新的 C++ 代码
- 格式化现有 C++ 代码
- 代码风格统一

---

### 2. cpp-coding-standards (C++ 编码规范)

基于 C++ Core Guidelines 的现代 C++ (C++17/20/23) 编码规范。适用于编写、审查或重构 C++ 代码时强制执行现代、安全和惯用的实践。

**核心原则**：
- RAII 资源管理
- 默认不可变性
- 类型安全
- 表达意图清晰
- 最小化复杂度

**适用场景**：
- 编写新的 C++ 代码
- 审查或重构现有 C++ 代码
- 在 C++ 项目中做出架构决策
- 在语言特性之间做出选择

---

### 3. dev-plan-manager (开发计划管理器)

创建和管理项目开发计划，支持任务分解、进度跟踪、燃尽图统计。

**核心功能**：
- 创建开发计划文档
- 任务分解（Phase → Task）
- 进度更新和统计
- 燃尽图数据记录

**适用场景**：
- 新项目启动时制定开发计划
- 功能开发前的任务分解
- 迭代规划和进度管理
- 任务完成后的进度更新

---

### 4. docx (Word 文档处理)

创建、读取、编辑和操作 Word 文档 (.docx 文件) 的技能。

**主要功能**：
- 使用 `docx-js` 创建新文档
- 解压 → 编辑 XML → 重新打包的工作流编辑现有文档
- 支持表格、图片、超链接、脚注、目录等
- 跟踪修订和注释处理
- 文档验证和修复

**适用场景**：
- 生成报告、备忘录、信函、模板
- 提取或重组 .docx 文件内容
- 插入或替换文档中的图片
- 查找和替换操作

---

### 5. encoding-converter (文件编码转换)

在 GBK、UTF-8、UTF-8 BOM 等编码之间转换文件编码。解决项目中的编码问题、乱码字符或编译错误。

**支持编码**：
- UTF-8 (无 BOM) - 推荐用于跨平台项目
- UTF-8 with BOM - Windows 默认
- GBK/GB2312 - 中文 Windows 传统编码
- UTF-16 LE

**适用场景**：
- 解决 Qt/C++ 项目中的混合编码问题
- 为需要 BOM 的编辑器添加 BOM
- 为遗留系统转换 UTF-8 到 GBK

---

### 6. find-skills (技能发现与安装)

帮助用户发现和安装 agent skills。当用户想要扩展 AI 助手的能力时使用此技能。

**主要功能**：
- 使用 `npx skills find` 搜索可用技能
- 使用 `npx skills add` 安装技能
- 检查和更新已安装的技能

**适用场景**：
- 用户问"怎么做某事"
- 用户说"找一个能做某事的技能"
- 用户想扩展 AI 助手的能力

---

### 7. function-refactoring (函数重构)

重构过长函数，拆分为职责单一的小函数。适用于函数超过100行、圈复杂度过高、难以测试和维护的场景。

**重构流程**：
1. 评估函数质量
2. 识别逻辑分段
3. 提取上下文结构体
4. 拆分子函数
5. 消除魔数
6. 重构主函数

**核心原则**：一个函数只做一件事，每个子函数控制在50行以内。

---

### 8. planning-and-task-breakdown (规划和任务分解)

将工作分解为小的、可验证的任务，并具有明确的验收标准。

**核心流程**：
1. 进入规划模式
2. 识别依赖关系图
3. 垂直切片
4. 编写任务
5. 排序和检查点

**适用场景**：
- 有规格说明需要分解为可实现单元
- 任务太大或太模糊无法开始
- 工作需要跨多个代理或会话并行化
- 需要向人类传达范围

---

### 9. requirements-analysis (需求分析)

诊断软件项目中的需求级问题，帮助区分表面需求与实际问题，发现真实约束。

**诊断状态**：
- RA0: 无问题陈述
- RA1: 方案优先思维
- RA2: 模糊需求
- RA3: 隐藏约束
- RA4: 范围蔓延预防
- RA5: 需求验证完成

**适用场景**：
- 新项目需求探索
- 需求澄清和验证
- 约束发现
- 范围定义

---

### 10. task-planning (任务规划)

有效规划和组织软件开发任务。处理任务分解、用户故事、验收标准和待办事项管理。

**核心功能**：
- 编写用户故事 (INVEST 原则)
- Epic → Story → Task 分解
- MoSCoW 优先级排序
- Sprint 规划

**适用场景**：
- 功能开发：分解新功能为小任务
- Sprint 规划：选择包含在 sprint 中的工作
- 待办事项整理：清理待办事项并设置优先级

---
