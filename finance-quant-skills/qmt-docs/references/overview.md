# QMT 系统概述

QMT（极速策略交易系统）是内置 Python 3.6 环境的量化交易平台，提供行情数据获取与交易下单两大核心功能。

## 核心特性

| 特性 | 说明 |
|------|------|
| Python 版本 | 3.6 |
| 支持库 | pandas、numpy、talib 等 |
| 编码 | GBK（必须首行声明） |
| 应用场景 | 指标计算、策略编写、回测、实盘 |

## 两种交易模式

### 回测模式

遍历历史 K 线数据，模拟交易过程。

| 项目 | 说明 |
|------|------|
| 数据源 | 本地历史数据 |
| subscribe 参数 | `False` |
| 撮合 | 虚拟撮合 |

**撮合规则：**
- 指定价格在 K 线高低点间 → 按指定价格
- 超出高低点范围 → 按收盘价
- 数量大于可用 → 按可用数量

### 实盘模式

盘中接收实时行情，发送真实交易信号。

| 项目 | 说明 |
|------|------|
| 数据源 | 实时推送 |
| subscribe 参数 | `True` |
| 撮合 | 真实交易所 |

**账号类型：**
- 模拟账号：QMT 模拟柜台
- 真实账号：连接真实交易所

**撮合规则：**
- 价格笼子：不能超过当前价 2%
- 数量超限：废单

## 三种运行机制

| 机制 | 触发方式 | 回测支持 | 实盘支持 | 典型场景 |
|------|---------|:--------:|:--------:|---------|
| **handlebar** | 逐 K 线 | ✅ | ✅ | 模拟 K 线、回测 |
| **subscribe** | 事件驱动 | ❌ | ✅ | 分笔高频交易 |
| **run_time** | 定时触发 | ❌ | ✅ | 定时监控 |

### handlebar（逐 K 线）

每根 K 线触发一次，最常用机制。

- 回测：历史 K 线从左向右逐根触发
- 实盘：分笔数据到达时，满足时间间隔触发

### subscribe（事件驱动）

新分笔到达时触发回调，适合高频策略。

- 仅支持实盘
- 分笔级别实时响应

### run_time（定时任务）

按固定时间间隔触发，适合监控场景。

- 仅支持实盘
- 固定间隔执行

## 场景选择指南

| 需求 | 运行机制 | 数据源 | 委托模式 |
|------|---------|--------|---------|
| 历史回测 | handlebar | 本地数据 | 逐 K 线 |
| 盘中模拟 K 线 | handlebar | 实时分笔 | 逐 K 线 |
| 分笔高频交易 | subscribe | 实时推送 | 立即下单 |
| 定时监控 | run_time | 定时查询 | 立即下单 |
